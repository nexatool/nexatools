# Security Policy

## Supported Versions

NexaTools is a continuously deployed web application. The version running at [nexatools.io](https://www.nexatools.io) is always the latest version, and it is the only version that receives security updates. There are no separate long-term-support or legacy versions to maintain.

| Version | Supported          |
| ------- | ------------------ |
| Latest (production, nexatools.io) | ✅ |
| Older commits / forks | ❌ |

If you are running a self-hosted fork of this repository, you are responsible for keeping it updated with the latest changes from `main`.

## Reporting a Vulnerability

If you discover a security vulnerability in NexaTools, please report it responsibly rather than disclosing it publicly.

**Please email:** contact@nexatools.io

When reporting, please include as much of the following as possible to help us triage quickly:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The tool or page affected (e.g. Resume Builder, Invoice Generator)
- Any relevant screenshots, logs, or proof-of-concept code
- The potential impact, as you understand it

Please **do not** open a public GitHub issue for security vulnerabilities. Use the email contact above instead.

## Responsible Disclosure Policy

We ask that security researchers and users:

- Give us a reasonable opportunity to investigate and address a reported vulnerability before disclosing it publicly.
- Avoid accessing, modifying, or deleting data that does not belong to you.
- Do not perform testing that could degrade the availability of NexaTools for other users (e.g. denial-of-service testing).
- Act in good faith and avoid privacy violations, data destruction, or service disruption while investigating an issue.

We are committed to working with anyone who reports a vulnerability in good faith and will not pursue legal action against researchers who follow this policy.

## Response Time

NexaTools is maintained on a best-effort basis. We aim to:

- Acknowledge new vulnerability reports as soon as possible after receiving them
- Investigate and assess the severity of the issue
- Work on a fix and keep the reporter updated on progress where possible

As response times depend on maintainer availability, we cannot guarantee fixed SLAs, but security reports are treated as a priority over other work.

## Scope

This policy covers the NexaTools website ([nexatools.io](https://www.nexatools.io)) and the code contained in this repository. Issues related to third-party services or infrastructure outside of our control should be reported directly to the relevant provider.

---

Thank you for helping keep NexaTools and its users safe.
